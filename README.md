# TC Lingo

検索練習・構造化された出力添削・同一テーマの反復練習、この3つに絞った語学学習アプリです。単語帳的なドリルや選択式クイズの代わりに、「自分の言葉で書く → AIに添削してもらう → 同じ問いにもう一度挑戦する」というループを繰り返すことを目的にしています。

tik-choco ファミリーの1アプリで、バックエンドを持たず、ユーザー自身のLLM APIキー(OpenAI互換エンドポイント)を使って動作します。

## セットアップ

```sh
npm install
npm run dev
```

`http://localhost:5173/`(埋まっていれば別ポート)で開発サーバーが起動します。

```sh
npm run build     # 本番ビルド(tsc -b && vite build)
npm run preview   # ビルド結果のプレビュー
```

## 使い方

1. **設定タブ**で学習言語(例: English)と母語(例: Japanese)、LLM接続先(baseUrl / APIキー / モデル名)を設定します。
2. **練習タブ**でAIにトピックを提案してもらうか、自分でトピックを入力し、自由に書いて「AIに添削してもらう」を押します。AIは「修正版」「理由」「再回答問題」の3点を返し、その場で言い直せます。
3. 添削結果から「間違いをカード化」すると、間違えた表現がSRSカードとして**カードタブ**の復習デッキに追加されます。
4. **復習タブ**では選択肢のない検索練習(思い出してから答えを見る形式)でカードを復習します。
5. 同じトピックは最大3回(初回／同日の改善版／後日の再挑戦)記録され、**履歴タブ**でラウンド間の差分を確認できます。

## LLM接続

接続情報(baseUrl / APIキー / モデル / temperature)は tik-choco ファミリーで共有される `tc-shared-llm-config-v1` に保存されます。同一オリジンで動く他の tik-choco アプリ(tc-translate など)で設定済みの接続があれば、TC Lingo でもそのまま選択できます。

## tik-choco ファミリーとの連携

同一オリジンにデプロイされた他のアプリと `localStorage` ベースで連携するための共有コントラクト(`src/lib/sharedBus.ts` / `appManifest.ts` / `llmConfig.ts`)を [`protocol`](../protocol) リポジトリからベンダリングしています。詳細は `CLAUDE.md` を参照してください。
